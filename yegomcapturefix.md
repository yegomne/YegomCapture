# YegomCapture (예곰 캡쳐) 에이전트별 작업 내역 정리

본 문서는 오늘 진행된 YegomCapture의 주요 기능 추가, 버그 수정 및 배포 파이프라인 개선 사항을 각 담당 에이전트별로 분류하여 정리한 내용입니다.

---

## 👩‍💻 유부장 (개발 파트너) - 코어 로직 & 신규 기능 개발
**1. 화면 캡쳐 후 그리기(편집) 기능 구현**
- 캡쳐된 이미지를 미리보기 하는 창에서 마우스를 드래그하여 빨간색 사각형을 그리고, 이를 캡쳐 이미지와 함께 저장할 수 있는 기능 추가.

**2. 업데이트 알림 끄기(무시) 기능 개발**
- 프로그램 시작 시 노출되는 '업데이트 알림' 팝업에 "현재 버전에 대한 알림 다시 보지 않기" 체크박스 UI 추가.
- 체크 시 `config.json`의 `ignored_update_version`에 해당 버전을 기록하여 다음번 실행부터는 동일 버전 팝업이 뜨지 않도록 예외 처리. (단, 더 높은 신규 버전 배포 시 정상 알림 작동)

**3. 개발 오답노트 (SKILL 업데이트)**
- 신규 기능 추가 시, 랜딩페이지 홍보 텍스트 및 배포 스크립트 정규식 치환 영역까지 고려하여 이과장(배포 담당)과 파이프라인 연동성을 크로스체크하도록 행동 지침 숙지.

**4. 미리보기 창 동적 스케일링(크기 조절) 최적화 및 좌표 매핑**
- 고해상도(4K 등) 모니터 및 윈도우 크기 조절 시 이미지가 잘리거나 여백이 생기는 현상 해결 (`setWidgetResizable(True)` 및 `paintEvent` 오버라이딩 적용).
- 이미지가 화면 크기에 맞춰 비율을 유지하며 동적 렌더링되도록 구현.
- 렌더링 화면에 그려지는 편집 상자(빨간 네모)의 좌표를 실제 원본 이미지 픽셀 해상도에 정확하게 매핑하여 그리기 로직 정확도 최적화.

**5. 단축키 먹통 현상(LowLevelHooksTimeout) 근본적 해결**
- 윈도우 보안 업데이트나 시스템 부하 시 `keyboard` 모듈의 저수준 후킹(Low-Level Hook)이 강제로 해제되어 단축키가 작동하지 않던 고질적 버그 해결.
- 외부 라이브러리(`keyboard`) 대신 윈도우 네이티브 API(`ctypes.windll.user32.RegisterHotKey`)를 직접 호출하는 방식으로 `HotkeyThread` 전면 재작성.
- 백신(Windows Defender) 오진단을 방지하고 어떤 상황에서도 절대 풀리지 않는 안정적인 전역 단축키 수신 구조 확보 완료.

## 🚀 이과장 (인프라/배포 요정) - 릴리즈 파이프라인 & 랜딩페이지 최적화
**1. 배포 파이프라인(스크립트) 버그 픽스 및 고도화**
- **다운로드 링크 404 에러 픽스:** 버전업 시 `index.html` 내 다운로드 링크 파일명이 이전 형식에 머물러 인식되지 않던 문제를 해결. (`scripts/bump_and_release.py`의 파일명 치환 정규식 로직 수정)
- **버전 뱃지 자동화 추가:** 다운로드 링크뿐만 아니라 랜딩페이지 상단 홍보용 버전 뱃지(예: `V1.0.4 업데이트`) 텍스트도 스크립트가 인식하여 일괄 교체하도록 추가 정규식 탑재 완료. (수동 이중 작업 방지)

**2. 랜딩페이지(웹) 및 가이드 문서 보완**
- 권한 에러 방지를 위해 `index.html`과 `guide.html`의 STEP 01 텍스트를 "setup.exe를 **우클릭하여 '관리자 권한으로 실행'** 해주세요"로 직관적으로 수정 후 즉시 배포.

**3. 인프라 오답노트 (SKILL 업데이트)**
- 자동 배포 스크립트 실행 이후에도, 랜딩페이지에 과거 버전의 흔적(배너 등)이 남아있지 않은지 최종 모니터링을 거치도록 운영 지침 숙지.

**4. v2.4 배포 파이프라인 원스톱 통과 및 인코딩 이슈 우회**
- 스케일링 패치가 포함된 v2.4 버전을 `bump_and_release.py` 스크립트를 통해 자동화 배포 완료 (버전 갱신, 빌드, GitHub 릴리즈, Git Push 적용).
- 윈도우 환경 터미널(PowerShell)의 인코딩 문제(`UnicodeEncodeError`)를 `$env:PYTHONIOENCODING="utf8"` 환경 변수 설정을 통해 우회하여 파이프라인 무중단 실행.
