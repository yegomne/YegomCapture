# 🚀 캡쳐 프로그램 배포 체크리스트 및 주의사항

## ⚠️ 설치 파일(Inno Setup) 빌드 시 주의사항 (좀비 프로그램 방지)
프로그램을 업데이트할 때 이름(`AppName`)이나 설치 폴더명(`DefaultDirName`)을 마음대로 바꾸면, 윈도우 OS는 "어? 이건 완전히 새로운 다른 앱이네?" 라고 착각해버립니다.
그래서 예전 버전이 안 지워지고 남아있다가 컴퓨터를 켤 때 두 개가 동시에 켜지는 무시무시한 일이 발생합니다!

### ✅ 다음부터는 이렇게 하시면 됩니다!
1. **기본 정보 유지:** `AppName` (예: 예곰 캡쳐 프로그램)과 `DefaultDirName` (예: {pf}\YegomCapture)은 가급적 처음에 픽스한 대로 쭈~욱 밀고 나가주세요!
2. **AppId 고정 (권장):** 만약 이름을 꼭 바꿔야 겠다면, Inno Setup 스크립트에 `AppId={{고정된-고유-GUID}}` 값을 부여해서, 윈도우가 "이름은 달라도 같은 놈이구나!" 하고 덮어쓰도록 유도해야 합니다.
3. **이름/경로를 바꿨다면?**
   - 다운로드 페이지에 **"꼭 기존 버전(간편 캡쳐 프로그램)을 제어판에서 삭제하신 후 새 버전을 설치해주세요!"** 라고 크게 공지해야 합니다.
   - 혹은 설치 파일 짤 때 기존 레지스트리를 부숴버리는(!) 강제 삭제 로직을 스크립트에 추가해야 합니다.

배포 전 이 문서를 한 번만 읽어보셔도 대참사를 막을 수 있습니다! 파이팅! 🚀💸

## ⚠️ V3/Defender 보안 경고 안내 및 다운로드 이탈 방어
1. **설치 시 팝업 주의점:** InnoSetup이나 PyInstaller로 빌드한 실행 파일(.exe)은 윈도우 디펜더나 보안 백신(V3 등)에서 초기 실행 시 "악성/의심 파일"로 오진되어 스마트 스크린 경고 알림(PC 보호창)을 띄우는 경우가 잦습니다. (코드 사인 미적용 시 특히 심함)
2. **랜딩페이지 조치사항:** 다운로드 버튼 하단이나 설치 안내 가이드 영역('STEP' 섹션)에, 놀란 고객들이 바로 창을 닫아버리지 않고 안심하고 설치를 1~2단계 진행할 수 있도록 **[방어용 안심 문구]**를 반드시 노출해야 합니다.
3. **디자인 키포인트:** '경고(Red)' 느낌보다 '안전(Amber/Blue)' 및 '방패' 아이콘 위주의 부드러운 UI로 설계하여 유저의 불안도를 낮추어 설치 전환율(CVR) 하락을 막아야 합니다.
